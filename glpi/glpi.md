# Запускаем cron
Для стабильной работы GLPI (сбор почты, отправка уведомлений, проверка SLA) встроенный планировщик автоматических действий необходимо перевести в режим работы операционной системы (CLI mode).  
По умолчанию используется режим GLPI, который срабатывает только тогда, когда пользователи активно пользуются веб-интерфейсом.
#### Шаг 1. Переключение режима в веб-интерфейсе GLPI
Переведите нужные или все автоматические действия на системный планировщик:  
1. Перейдите в Настройки (Setup) → Автоматические действия (Automatic actions).
2. Выберите критически важную задачу (например, mailgate для сбора почты или queuednotification для отправки уведомлений).
3. Измените Режим выполнения (Run mode) с GLPI на CLI.
4. Повторите процедуру для остальных важных системных задач.
#### Шаг 2. Настройка системного Crontab
Команда запуска должна выполняться строго от имени пользователя веб-сервера (например, www-data, apache или nginx), чтобы не нарушить права доступа к файлам.   
Откройте планировщик crontab под пользователем вашего веб-сервера:
```
crontab -u www-data -e
```
Добавьте в конец файла следующую строку (проверяет задачи каждую минуту)
```
* * * * * /usr/bin/php /var/www/glpi/front/cron.php &> /dev/null
```
